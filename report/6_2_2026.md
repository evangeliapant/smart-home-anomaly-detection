# Project Note: Unsupervised Routine Discovery and Anomaly Detection in Smart Homes

## Motivation and Literature Takeaways
Most smart-home activity recognition methods rely on supervised learning with manually labeled activities, which is costly and impractical for typical consumer smart-home deployments. Unsupervised approaches aim to learn recurring patterns (routines) directly from unlabeled sensor event streams and to identify deviations from learned normal behavior. Yin (2015) presents an unsupervised method to discover daily routines and activities in smart homes without requiring a pre-defined labeled activity set, supporting the feasibility of learning structure directly from raw behavior traces. Soulas et al. (2015) emphasize that habits and their periodicity/variability can be extracted from smart-home sensor data and that changes in regular behavior are meaningful for monitoring. Huynh et al. (2008) demonstrate that daily routines can be modeled as combinations of latent patterns, motivating cluster/topic-like representations for routine discovery. Cook (2014) highlights smart-home data mining challenges such as noisy event streams, sparse observations, and the need for robust representations. Finally, process discovery from sensor event logs (Janssen et al., 2020) motivates treating smart-home sensor streams as event logs that can be segmented and summarized, aligning with a semi-industrial pipeline view.

Based on these findings, this project focuses on transforming unlabeled sensor event streams into structured representations (fixed windows and sessions), discovering routine clusters, and detecting anomalies using unsupervised models. An interpretation layer maps clusters and anomalies into human-readable routine descriptions and safety-relevant deviation explanations, enabling demonstration of automation triggers without manual annotation.

## Repository Structure and Purpose
- `data/raw/`: original event logs (unmodified).
- `data/processed/`: model-ready feature tables produced by the pipeline.
- `src/preprocessing/`: parsing and cleaning of event logs (timestamp normalization, sorting, deduplication).
- `src/features/`: segmentation (fixed windows + sessionization) and feature engineering (sensor counts, time-of-day encoding, density, duration).
- `src/models/`: unsupervised learning modules:
  - `clustering.py`: routine discovery (e.g., KMeans/DBSCAN variants).
  - `anomaly.py`: anomaly detection (e.g., Isolation Forest; later autoencoder).
- `src/automation/`: rule-based simulator mapping learned patterns/anomalies to automation actions/alerts.
- `scripts/`: reproducible runs (pipeline execution, sessionization experiments, demo runs).
- `notebooks/`: analysis and interpretation notebooks (visualization, cluster labeling, anomaly case studies).
- `outputs/`: saved figures and trained models for reproducibility.
- `report/`: thesis draft and research notes.

This structure supports both a publishable research prototype (clear experiments, reproducibility, interpretation) and a semi-industrial ML system style (modular pipeline, automation-ready outputs).
